## TL;DR

Tempo is an EVM-compatible Layer 1 blockchain launched on mainnet March 18, 2026, designed as high-throughput infrastructure for stablecoin payments and agentic commerce. Co-incubated by Stripe and Paradigm, it differentiates from general-purpose chains by prioritizing predictable low fees (paid in stablecoins via an integrated AMM), sub-second finality, and native account abstraction—making it a specialized "rail" for real-world payment flows like remittances, payroll, and AI agent transactions rather than a native stablecoin issuer. With $500M in Series A funding and partnerships including Visa, Mastercard, and Klarna, Tempo positions itself as the "decentralized, internet-scale SWIFT" for stablecoins, but its extreme recency limits on-chain visibility. [Tempo](https://tempo.xyz/blog/mainnet)

## 2. Product & Technical Stack

Tempo's stack centers on payments optimization: no native gas token (fees in any stablecoin via TIP-20 AMM), native account abstraction (passkeys, sponsored fees, batching), and MPP for autonomous agents. Key modules from docs/blog:

| Module                          | Description                                                                                          | Key Features                                                                                                                                                                                                                                              |
| ------------------------------- | ---------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TIP-20 Token Standard           | Native ERC-20 extension for payments. [Tempo Blog](https://tempo.xyz/blog/tip20)                     | Stablecoin-denominated fees (AMM converts user stablecoin to validator preference); native mint/burn/restrictions; yield distribution; transfer memos (ISO 20022 compliant).                                                                              |
| Machine Payments Protocol (MPP) | Open standard co-authored with Stripe for agent-to-machine payments. [MPP.dev](https://mpp.dev/)     | "Sessions": one-time authorization reserves funds; streams micro-payments (e.g., API calls) aggregated into single settlement. Supports stablecoins/cards/Lightning. Directory of 100+ services (Alchemy data, Dune queries, fal.ai models, Browserbase). |
| Account Abstraction             | Protocol-level (no smart contracts needed). [Tempo Blog](https://tempo.xyz/blog/account-abstraction) | Passkey sign-in; fee sponsorship; scoped access keys; batch/parallel/scheduled txns.                                                                                                                                                                      |
| Compliance & Policies           | TIP-403 for role-based controls. [Tempo Blog](https://tempo.xyz/blog/compliance-controls)            | Mint/freeze/burn delegation; audit logs; default-deny for policies.                                                                                                                                                                                       |
| Payments Flows                  | Mint/redeem via partners; no central reserves (third-party issuers).                                 | Payroll batching, remittances, embedded finance; on-chain FX roadmap.                                                                                                                                                                                     |

Stablecoin Type: Rail for fiat/RWA-backed third-party stablecoins (e.g., Klarna's planned issuance). No native Tempo stablecoin; peg via issuer reserves (not managed by Tempo). Integrates wallets (passkeys), DEXs (0x Swap API live [TradingView](https://www.tradingview.com/news/coindar:5cf9175dd094b:0-0x-launches-swap-api-support-on-tempo-mainnet/)), custody (UBS testnet). SDKs/RPCs public; EVM-compatible. [Docs](https://docs.tempo.xyz/)

This setup excels for high-volume B2B (60% stablecoin volume) but relies on issuer transparency for peg stability.
