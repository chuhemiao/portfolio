## 2. Product & Technical Architecture

### Hybrid Execution Engine

Pacifica's architecture represents a fundamental rethinking of decentralized exchange design, optimizing for one thing: the best trading experience. The platform employs a hybrid model that separates matching logic from settlement to achieve maximum performance without compromising custody.

Off-Chain Matching Engine:

- Sub-10ms Round-Trip Latency: The matching engine operates off-chain for speed, delivering API latency competitive with tier-1 centralized exchanges like Binance and FTX

- High Throughput: Built from scratch in Rust for full control and precision, enabling institutional-grade order processing

- Order Book Model: Professional-grade order book with deep liquidity maintained through sophisticated market maker incentives

On-Chain Settlement:

- Non-Custodial Guarantees: Settlement and custody remain fully on-chain via Solana smart contracts

- Transparent Verification: Users hold their keys at all times; all positions and balances verifiable on-chain

- Bankruptcy-Remote Design: Sophisticated cold vault/hot wallet separation ensures user funds remain secure even in catastrophic scenarios

For context, most decentralized exchanges operate in the hundreds of milliseconds, while Pacifica's latency is competitive with tier-1 centralized exchanges. This performance advantage is critical for institutional traders, market makers, and algorithmic strategies. [citation](https://docs.pacifica.fi/pacifica/long-term-vision-and-roadmap)

### Advanced Order Types & Trading Features

Pacifica offers a comprehensive suite of order types designed for professional traders:

- Market Orders: Instant execution at best available price

- Limit Orders: Specify exact entry/exit prices

- Stop-Loss & Take-Profit: Automated risk management

- Post-Only Orders: Ensure maker fee tier

- Reduce-Only Orders: Close positions without adding exposure

- Self-Trade Prevention: Built-in safeguard that automatically cancels resting orders when they would match against the same account

Leverage & Margin:

- Up to 50x leverage on perpetual futures

- Unified margin accounts (Phase 2 roadmap) will enable multi-collateral margin across perpetual and spot positions

- Real-time cross-margined risk calculation with robust liquidation mechanics

Funding Rate Mechanism:

- Hourly funding rate settlements (24x daily vs. traditional 8-hour systems)

- Funding rates capped at 4% to prevent extreme imbalances

- Calculated based on gap between orderbook impact prices and oracle prices, keeping perp markets tightly anchored to spot

### Oracle & Price Feed Infrastructure

Pacifica employs a robust oracle system to ensure accurate pricing and fair liquidations:

- Oracle Refresh Rate: Every 3 seconds

- Funding Rate Recalculation: Every 5 seconds

- Settlement Frequency: Hourly

- Price Sources: Multiple oracle providers to prevent single-point-of-failure risk

### Security Architecture: Cold Vault + Hot Wallet Model

Pacifica has implemented an institutional-grade security model that addresses the core challenge of high-performance perpetual DEXs: managing withdrawal authority while maintaining non-custodial guarantees.

Cold Vault (Squads Protocol Multisig):

- The bulk of user capital resides in a cold vault secured by a multi-signature smart contract

- Governed by a decentralized council that is geo-distributed across multiple continents

- Squads Protocol: The first formally verified program on Solana, meaning its code has undergone mathematical proofs of correctness beyond traditional auditing

- Secures over $$10 billion in assets across the Solana ecosystem

- Primary function: Replenish the hot wallet through programmatic spending limits

Hot Wallet:

- Facilitates instant withdrawals for seamless user experience

- Maintains minimal balance necessary for operational liquidity

- Automatically replenished from cold vault via programmatic controls

This architecture contrasts with most perpetual DEXs that operate with a unified smart contract where all user funds reside in a single upgradeable contract. In those systems, if administrators can upgrade the logic contract, they possess the ability to modify withdrawal permissions—a significant centralization risk. [citation](https://docs.pacifica.fi/trading-on-pacifica/fund-security)

## 3. Smart Contracts & Security

### Smart Contract Architecture

Pacifica's smart contracts are built on Solana using native Rust programs, optimized for high-performance perpetual futures trading. The platform employs a modular architecture separating core functions:

- Perpetual Contract: Handles position management, margin calculations, and liquidations

- Settlement Contract: Manages on-chain settlement and custody

- Oracle Integration: Connects to multiple price feed providers

- Multisig Vault: Squads Protocol integration for cold storage

### Security Audits

BlockSec Audit:

- Pacifica's perpetual contracts have been audited by BlockSec, a leading blockchain security firm

- Audit report publicly available covering smart contract security, access controls, and economic mechanisms

- Scope included perpetual futures logic, margin calculations, liquidation mechanics, and oracle integration [citation](https://coinlaunch.space/projects/pacifica/)

Security Measures:

- Non-Custodial Architecture: Users maintain full custody of funds via wallet signatures

- Separate Wallet System for AI Agents: AI trading tools operate with isolated permissions to prevent unauthorized access

- Hardware Wallet Support: Integration with Ledger and other hardware wallets for enhanced security

- Upgrade Timelocks: Changes to contracts require waiting periods, giving users time to react

- Multi-Signature Controls: Critical operations require multiple authorized signatures

- Bug Bounty Program: Incentivizes security researchers to find and report vulnerabilities responsibly [citation](https://www.journee-mondiale.com/en/pacifica-fi-review-2025-complete-guide-to-solanas-rising-perpetual-dex/)

### Risk Surface

Smart Contract Risk:

- Despite professional audits, undiscovered bugs could theoretically exist in the code

- Mitigation: Formal verification for critical functions, timelocked upgrades, bug bounty program

Oracle Risk:

- Reliance on external price feeds for liquidations and funding rates

- Mitigation: Multiple oracle providers, 3-second refresh rate, price deviation monitoring

Solana Network Dependency:

- Platform cannot trade during Solana downtime

- Mitigation: Funds remain secure in smart contracts during network outages; Solana's reliability has improved significantly in 2025-2026

Liquidity Risk:

- Liquidity depth still growing compared to established CEXs

- Mitigation: Market maker program, sophisticated incentive design, tight spreads maintained
