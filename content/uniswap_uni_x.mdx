## 2. Protocol Architecture and AMM Design

### Core AMM Mechanics

Uniswap's constant product formula `x * y = k` remains the foundation across all versions, enabling automated market making without order books. Each pool maintains reserves of two tokens with prices determined by their ratio.

### Architectural Evolution Comparison

| Feature         | v2                   | v3                                | v4                     |
| --------------- | -------------------- | --------------------------------- | ---------------------- |
| Liquidity Model | Uniform distribution | Concentrated ranges               | Concentrated + Hooks   |
| Position Tokens | Fungible ERC-20      | Non-fungible NFTs                 | Singleton-managed      |
| Fee Tiers       | Fixed 0.3%           | Multiple (0.01%, 0.05%, 0.3%, 1%) | Customizable via hooks |
| Gas Efficiency  | Baseline             | ~2x improvement                   | ~99% reduction vs v3   |
| Customization   | None                 | Limited                           | Unlimited via hooks    |

### v4 Singleton Architecture

The revolutionary singleton design consolidates all pools into a single contract, dramatically reducing gas costs and enabling unprecedented customization through hooks. As of January 2026, 23,138 hooks have been initialized across the ecosystem, demonstrating strong developer adoption. [Uniswap](https://x.com/Uniswap/status/2008577775687659542)

### Uniroute Optimization Engine

Recently deployed Uniroute represents a complete rebuild of swap routing, delivering superior pricing on:

- 56% of Ethereum swaps
- 62% of Base swaps
- 67% of Arbitrum swaps
- 76% of Unichain swaps

Additionally, Uniroute provides 600ms+ faster execution for complex routes. [Uniswap](https://x.com/Uniswap/status/2015090795187949642)
